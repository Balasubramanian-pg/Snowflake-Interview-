# Understanding Deadlocks in Hybrid Transactions

**Description:**
Deadlocks are critical system failures that occur due to improper resource allocation and synchronization. They happen when processes wait for each other to release resources, leading to system crashes, data corruption, and downtime. Understanding deadlocks is crucial for designing efficient and reliable concurrent systems.

In a multi-threaded environment, two processes (P1 and P2) need to access two shared resources (R1 and R2). If P1 locks R1 and waits for R2, while P2 locks R2 and waits for R1, a deadlock occurs.
