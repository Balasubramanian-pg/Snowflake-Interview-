# 85.6. Totals
> Accurate totals are crucial for making informed decisions in various aspects of life, including finance, business, and personal planning.
> 
## What are Totals in Pharma Data?

In pharmaceutical analytics, **totals** represent complete summations of financial, operational, or clinical data. Unlike averages that summarize central tendency, totals provide **absolute quantities** essential for:
- **Financial reconciliation** (total claims paid, total rebates owed)
- **Inventory management** (total units in stock, total value of inventory)
- **Regulatory reporting** (total controlled substances dispensed, total adverse events)
- **Contract compliance** (total purchases vs. contract minimums)

## Use Case 1: Claims Financial Reconciliation (837/835)

**Scenario**: Calculate complete financial totals for claims processing reconciliation, including all cost components.

```sql
-- Create comprehensive claims financial data
CREATE OR REPLACE TABLE claims_financial_details (

    ndc_code VARCHAR(11),

    patient_copay DECIMAL(10,2),
    patient_deductible DECIMAL(10,2),
    patient_coinsurance DECIMAL(10,2),
    plan_paid_amount DECIMAL(10,2),

-- TOTALS QUERY: Complete financial reconciliation with all components
WITH component_totals AS (
    -- Calculate totals for each financial component (like summing all line items)
    SELECT 
        payer_id,
        
        -- Core payment totals
        SUM(billed_amount) AS total_billed,
        SUM(allowed_amount) AS total_allowed,
        SUM(patient_copay + patient_deductible + patient_coinsurance) AS total_patient_responsibility,
        SUM(plan_paid_amount) AS total_plan_paid,
        
        -- Adjustment totals (can be positive or negative)
        SUM(contractual_adjustment) AS total_contractual_adjustments,
        SUM(sequestration_adjustment) AS total_sequestration_cuts,
        SUM(penalty_amount) AS total_penalties,
        SUM(bonus_amount) AS total_bonuses,
        
        -- Fee totals (often forgotten in quick calculations)
        SUM(dispensing_fee) AS total_dispensing_fees,
        SUM(administrative_fee) AS total_admin_fees,
        SUM(shipping_fee) AS total_shipping,
        SUM(sales_tax) AS total_tax,
        
        -- Verification: Billed should equal sum of components
        SUM(allowed_amount + contractual_adjustment + sequestration_adjustment) AS billed_reconstruction,
        
        -- Counts for context
        COUNT(*) AS total_claims,
        SUM(CASE WHEN claim_status = 'PAID' THEN 1 ELSE 0 END) AS paid_claims,
        SUM(CASE WHEN claim_status = 'DENIED' THEN 1 ELSE 0 END) AS denied_claims
        
    FROM claims_financial_details
    
    GROUP BY payer_id
),
grand_totals AS (
    -- Calculate grand totals across all payers (rollup)
    SELECT 
        'ALL_PAYERS' AS payer_id,
        SUM(total_billed) AS total_billed,
        SUM(total_allowed) AS total_allowed,
        SUM(total_patient_responsibility) AS total_patient_responsibility,
        SUM(total_plan_paid) AS total_plan_paid,
        SUM(total_contractual_adjustments) AS total_contractual_adjustments,
        SUM(total_sequestration_cuts) AS total_sequestration_cuts,
        SUM(total_penalties) AS total_penalties,
        SUM(total_bonuses) AS total_bonuses,
        SUM(total_dispensing_fees) AS total_dispensing_fees,
        SUM(total_admin_fees) AS total_admin_fees,
        SUM(total_shipping) AS total_shipping,
        SUM(total_tax) AS total_tax,
        SUM(billed_reconstruction) AS billed_reconstruction,
        SUM(total_claims) AS total_claims,
        SUM(paid_claims) AS paid_claims,
        SUM(denied_claims) AS denied_claims
    FROM component_totals
    
    UNION ALL
    
    SELECT * FROM component_totals
),
reconciliation_checks AS (
    SELECT 
        *,
        
        -- Critical reconciliation: Verify totals balance
        -- Rule: Allowed + Adjustments + Fees should reconstruct the financial flow
        total_allowed + total_contractual_adjustments + total_sequestration_cuts AS calculated_total_adjusted,
        
        -- Patient + Plan should equal Allowed (conservation of payment)
        total_patient_responsibility + total_plan_paid AS calculated_total_paid,
        
        -- Reconciliation variance (should be zero)
        total_allowed - (total_patient_responsibility + total_plan_paid) AS payment_variance,
        
        -- Total cost to payer (plan paid + all fees passed through)
        total_plan_paid + total_dispensing_fees + total_admin_fees + total_shipping AS total_payer_cost,
        
        -- Total revenue to pharmacy (patient responsibility + plan paid + fees - penalties)
        total_patient_responsibility + total_plan_paid + total_dispensing_fees + total_admin_fees + 
            total_shipping + total_tax - total_penalties + total_bonuses AS total_pharmacy_revenue,
        
        -- Data quality checks
        CASE 
            WHEN ABS(payment_variance) > 0.01 THEN 'RECONCILIATION_ERROR'
            WHEN total_billed <> billed_reconstruction THEN 'BILLED_RECONSTRUCTION_FAILED'
            ELSE 'BALANCED'
        END AS financial_integrity_status
        
    FROM grand_totals
)
SELECT 
    payer_id,
    total_claims,
    paid_claims,
    denied_claims,
    
    -- Core financial totals
    ROUND(total_billed, 2) AS total_billed_amount,
    ROUND(total_allowed, 2) AS total_allowed_amount,
    ROUND(total_patient_responsibility, 2) AS total_patient_payments,
    ROUND(total_plan_paid, 2) AS total_plan_payments,
    
    -- Adjustment totals (often overlooked)
    ROUND(total_contractual_adjustments, 2) AS total_write_offs,
    ROUND(total_sequestration_cuts, 2) AS federal_sequestration,
    ROUND(total_penalties, 2) AS total_penalty_charges,
    ROUND(total_bonuses, 2) AS total_quality_bonuses,
    
    -- Fee totals (silent cost drivers)
    ROUND(total_dispensing_fees, 2) AS total_dispensing_fees,
    ROUND(total_admin_fees, 2) AS total_administrative_fees,
    ROUND(total_shipping, 2) AS total_shipping_costs,
    ROUND(total_tax, 2) AS total_sales_tax,
    
    -- Composite totals
    ROUND(total_payer_cost, 2) AS complete_payer_expenditure,
    ROUND(total_pharmacy_revenue, 2) AS complete_pharmacy_revenue,
    
    -- Verification
    ROUND(payment_variance, 2) AS reconciliation_variance,
    financial_integrity_status AS data_quality_check,
    
    -- Critical alerts (silent failure detection)
    CASE 
        WHEN total_penalties > total_bonuses * 10 THEN 'PENALTY_HEAVY_CONTRACT_REVIEW'
        WHEN total_sequestration_cuts > total_allowed * 0.05 THEN 'SEQUESTRATION_IMPACT_HIGH'
        WHEN total_dispensing_fees > total_plan_paid * 0.10 THEN 'FEE_STRUCTURE_EXCESSIVE'
        ELSE 'COST_STRUCTURE_NORMAL'
    END AS financial_health_alert
    
FROM reconciliation_checks

ORDER BY 
    CASE WHEN payer_id = 'ALL_PAYERS' THEN 0 ELSE 1 END,
    total_billed DESC;
```

**Workflow Explanation**:

1. **Component Summation** - `SUM()` each financial field separately (identifying all components)
2. **Cross-Verification** - Reconstructs billed amount from allowed + adjustments (double-check calculation)
3. **Conservation Check** - Verifies patient + plan = allowed (payment conservation law)
4. **Grand Total Rollup** - Unions payer-level and all-payer totals (complete aggregation)
5. **Silent Failure Detection** - Identifies when penalties exceed bonuses or fees are excessive

## Use Case 2: Inventory Position Totals (EDI 852)

**Scenario**: Calculate complete inventory totals across all locations with valuation and reconciliation.

```sql
-- Create comprehensive inventory data
CREATE OR REPLACE TABLE inventory_positions (
    inventory_id VARCHAR(30),
    ndc_code VARCHAR(11),
    lot_number VARCHAR(20),
    expiration_date DATE,
    
    -- Location hierarchy
    warehouse_id VARCHAR(20),
    zone_id VARCHAR(10),
    bin_location VARCHAR(20),
    
    -- Quantity components
    on_hand_quantity INTEGER,
    reserved_quantity INTEGER, -- Committed to orders
    quarantined_quantity INTEGER, -- Quality hold
    in_transit_quantity INTEGER, -- Shipping between locations
    on_order_quantity INTEGER, -- Pending receipt
    
    -- Financial components
    unit_cost DECIMAL(10,4),
    average_landed_cost DECIMAL(10,4),
    replacement_cost DECIMAL(10,4),
    market_value DECIMAL(10,4),
    
    -- Status
    inventory_status VARCHAR(20), -- 'active', 'hold', 'expired', 'disposed'
    last_movement_date DATE,

('INV_004', '00093-2748-01', 'LOT_B001', '2025-12-01', 'WH_EAST', 'Q', 'Q-01-01',

('INV_005', '00093-1058-01', 'LOT_A000', '2024-02-01', 'WH_EAST', 'X', 'X-99-01',

-- In-transit between warehouses
('INV_006', '00555-0808-02', 'LOT_C001', '2026-03-01', 'IN_TRANSIT', 'T', 'TRUCK_001', 
 0, 0, 0, 1500, 0, 1.1000, 1.1500, 1.4500, 1.2500, 'active', '2024-03-16', 0);

-- TOTALS QUERY: Complete inventory valuation and reconciliation
WITH quantity_totals AS (
    -- Calculate quantity totals by status and availability
    SELECT 
        ndc_code,
        
        -- Total physical inventory (regardless of status)
        SUM(on_hand_quantity) AS total_on_hand,
        SUM(reserved_quantity) AS total_reserved,
        SUM(quarantined_quantity) AS total_quarantined,
        SUM(in_transit_quantity) AS total_in_transit,
        SUM(on_order_quantity) AS total_on_order,
        
        -- Available to promise (ATP) = On hand - Reserved - Quarantined
        SUM(on_hand_quantity - reserved_quantity - quarantined_quantity) AS total_available,
        
        -- Total inventory position (all sources)
        SUM(on_hand_quantity + in_transit_quantity + on_order_quantity) AS total_inventory_position,
        
        -- Status breakdowns
        SUM(CASE WHEN inventory_status = 'active' THEN on_hand_quantity ELSE 0 END) AS active_inventory,
        SUM(CASE WHEN inventory_status = 'hold' THEN on_hand_quantity ELSE 0 END) AS hold_inventory,
        SUM(CASE WHEN inventory_status = 'expired' THEN on_hand_quantity ELSE 0 END) AS expired_inventory,
        
        -- Location diversity
        COUNT(DISTINCT warehouse_id) AS warehouse_count,
        COUNT(DISTINCT lot_number) AS lot_count,
        
        -- Movement analysis
        SUM(CASE WHEN days_since_movement > 30 THEN on_hand_quantity ELSE 0 END) AS slow_moving_inventory,
        SUM(CASE WHEN days_since_movement > 90 THEN on_hand_quantity ELSE 0 END) AS dead_inventory
        
    FROM inventory_positions
    
    GROUP BY ndc_code
),
financial_totals AS (
    -- Calculate financial totals using appropriate cost basis
    SELECT 
        ndc_code,
        
        -- Cost basis totals (conservative, what we paid)
        SUM(on_hand_quantity * average_landed_cost) AS total_landed_cost,
        SUM(on_hand_quantity * unit_cost) AS total_unit_cost_value,
        
        -- Market value totals (current worth)
        SUM(on_hand_quantity * market_value) AS total_market_value,
        SUM(on_hand_quantity * replacement_cost) AS total_replacement_cost,
        
        -- Valuation by status (critical for write-offs)
        SUM(CASE WHEN inventory_status = 'expired' THEN on_hand_quantity * unit_cost ELSE 0 END) AS expired_inventory_value,
        SUM(CASE WHEN inventory_status = 'hold' THEN on_hand_quantity * unit_cost ELSE 0 END) AS quarantined_value,
        SUM(CASE WHEN inventory_status = 'active' THEN on_hand_quantity * market_value ELSE 0 END) AS active_market_value,
        
        -- In-transit valuation
        SUM(in_transit_quantity * average_landed_cost) AS in_transit_value,
        
        -- Reserved inventory value (committed revenue)
        SUM(reserved_quantity * market_value) AS reserved_revenue_potential
        
    FROM inventory_positions
    
    GROUP BY ndc_code
),
grand_totals AS (
    -- Calculate grand totals across all NDCs
    SELECT 
        'ALL_PRODUCTS' AS ndc_code,
        SUM(total_on_hand) AS total_on_hand,
        SUM(total_reserved) AS total_reserved,
        SUM(total_quarantined) AS total_quarantined,
        SUM(total_in_transit) AS total_in_transit,
        SUM(total_on_order) AS total_on_order,
        SUM(total_available) AS total_available,
        SUM(total_inventory_position) AS total_inventory_position,
        SUM(active_inventory) AS active_inventory,
        SUM(hold_inventory) AS hold_inventory,
        SUM(expired_inventory) AS expired_inventory,
        COUNT(DISTINCT warehouse_id) AS warehouse_count, -- Note: this needs adjustment for grand total
        SUM(lot_count) AS lot_count,
        SUM(slow_moving_inventory) AS slow_moving_inventory,
        SUM(dead_inventory) AS dead_inventory,
        SUM(total_landed_cost) AS total_landed_cost,
        SUM(total_unit_cost_value) AS total_unit_cost_value,
        SUM(total_market_value) AS total_market_value,
        SUM(total_replacement_cost) AS total_replacement_cost,
        SUM(expired_inventory_value) AS expired_inventory_value,
        SUM(quarantined_value) AS quarantined_value,
        SUM(active_market_value) AS active_market_value,
        SUM(in_transit_value) AS in_transit_value,
        SUM(reserved_revenue_potential) AS reserved_revenue_potential
    FROM quantity_totals qt
    JOIN financial_totals ft ON qt.ndc_code = ft.ndc_code
    
    UNION ALL
    
    SELECT 
        qt.ndc_code,
        total_on_hand,
        total_reserved,
        total_quarantined,
        total_in_transit,
        total_on_order,
        total_available,
        total_inventory_position,
        active_inventory,
        hold_inventory,
        expired_inventory,
        warehouse_count,
        lot_count,
        slow_moving_inventory,
        dead_inventory,
        total_landed_cost,
        total_unit_cost_value,
        total_market_value,
        total_replacement_cost,
        expired_inventory_value,
        quarantined_value,
        active_market_value,
        in_transit_value,
        reserved_revenue_potential
    FROM quantity_totals qt
    JOIN financial_totals ft ON qt.ndc_code = ft.ndc_code
),
inventory_metrics AS (
    SELECT 
        *,
        
        -- Inventory health ratios
        ROUND(active_inventory * 100.0 / NULLIF(total_on_hand, 0), 2) AS pct_active_inventory,
        ROUND(expired_inventory * 100.0 / NULLIF(total_on_hand, 0), 2) AS pct_expired_inventory,
        ROUND(slow_moving_inventory * 100.0 / NULLIF(total_on_hand, 0), 2) AS pct_slow_moving,
        
        -- Financial metrics
        ROUND(total_market_value - total_landed_cost, 2) AS unrealized_gain_loss,
        ROUND((total_market_value - total_landed_cost) / NULLIF(total_landed_cost, 0) * 100, 2) AS inventory_appreciation_pct,
        
        -- Write-off exposure
        ROUND(expired_inventory_value + quarantined_value, 2) AS at_risk_inventory_value,
        
        -- Turnover indicator
        ROUND(reserved_revenue_potential / NULLIF(total_market_value, 0) * 100, 2) AS pct_inventory_committed,
        
        -- Data quality check
        CASE 
            WHEN total_on_hand <> active_inventory + hold_inventory + expired_inventory 
            THEN 'QUANTITY_RECONCILIATION_ERROR'
            WHEN ABS(total_market_value - (active_market_value + expired_inventory_value + quarantined_value)) > 1
            THEN 'VALUE_RECONCILIATION_ERROR'
            ELSE 'INVENTORY_BALANCED'
        END AS inventory_integrity_status
        
    FROM grand_totals
)
SELECT 
    ndc_code AS product_identifier,
    
    -- Quantity totals (complete inventory position)
    total_on_hand AS total_units_in_warehouse,
    total_reserved AS units_committed_to_orders,
    total_quarantined AS units_quality_hold,
    total_in_transit AS units_between_locations,
    total_on_order AS units_pending_receipt,
    total_available AS units_available_to_sell,
    total_inventory_position AS total_supply_chain_position,
    
    -- Status breakdown totals
    active_inventory AS active_sellable_units,
    hold_inventory AS quarantined_units,
    expired_inventory AS expired_write_off_units,
    
    -- Financial totals (multiple valuation methods)
    ROUND(total_landed_cost, 2) AS inventory_cost_basis,
    ROUND(total_market_value, 2) AS inventory_market_value,
    ROUND(total_replacement_cost, 2) AS inventory_replacement_cost,
    unrealized_gain_loss AS appreciation_vs_cost,
    
    -- Risk exposure totals
    ROUND(expired_inventory_value, 2) AS expired_write_off_dollar,
    ROUND(quarantined_value, 2) AS quarantine_at_risk_dollar,
    at_risk_inventory_value AS total_at_risk_value,
    
    -- Operational totals
    warehouse_count AS locations_stocked,
    lot_count AS distinct_lots,
    slow_moving_inventory AS units_no_movement_30days,
    dead_inventory AS units_no_movement_90days,
    
    -- Performance ratios
    pct_active_inventory AS pct_sellable,
    pct_expired_inventory AS pct_write_off,
    pct_slow_moving AS pct_stagnant,
    inventory_appreciation_pct AS value_change_pct,
    
    -- Data quality
    inventory_integrity_status AS reconciliation_status,
    
    -- Business alerts (silent failure detection)
    CASE 
        WHEN expired_inventory_value > total_market_value * 0.05 THEN 'WRITE_OFF_EXCEEDS_5PCT_INVESTIGATION'
        WHEN slow_moving_inventory > total_on_hand * 0.30 THEN 'STAGNANT_INVENTORY_CRITCAL'
        WHEN pct_inventory_committed > 80 THEN 'OVERCOMMITTED_STOCK_RISK'
        ELSE 'INVENTORY_HEALTHY'
    END AS inventory_health_alert
    
FROM inventory_metrics

ORDER BY 
    CASE WHEN ndc_code = 'ALL_PRODUCTS' THEN 0 ELSE 1 END,
    total_market_value DESC;
```

**Workflow Explanation**:

1. **Component Quantities** - Sums each quantity type (on hand, reserved, quarantined, in-transit, on-order)
2. **Derived Totals** - Calculates ATP (available to promise) and total position from components
3. **Multi-Basis Valuation** - Calculates totals at landed cost, market value, and replacement cost
4. **Status Segregation** - Separates active, hold, and expired for write-off analysis
5. **Conservation Check** - Verifies on_hand = active + hold + expired (quantity conservation)

    manufacturer,
    ndc_code,
    
    -- Authorization portfolio totals
    total_authorizations AS total_auth_count,
    active_auths AS active_authorizations,
    expired_auths AS expired_authorizations,

    pending_auths AS pending_approval,
    
    -- Quantity flow totals
    total_units_authorized AS total_auth_units,
    total_units_dispensed AS units_filled_to_date,
    total_units_remaining AS units_still_available,
    active_remaining_inventory AS active_sellable_units,
    expired_unused_units AS expired_wasted_units,
    revoked_lost_units AS revoked_lost_units,
    
    -- Financial totals (critical for revenue recognition)
    ROUND(total_contract_value, 2) AS total_contract_potential,
    ROUND(total_realized_revenue, 2) AS revenue_recognized_to_date,
    ROUND(total_pending_revenue, 2) AS future_revenue_pipeline,
    ROUND(active_pipeline_value, 2) AS active_auth_unrealized_value,
    ROUND(expired_lost_revenue + revoked_lost_revenue, 2) AS total_revenue_leakage,
    
    -- Patient financial totals
    ROUND(total_patient_copay_liability, 2) AS total_copay_expected,
    ROUND(total_copay_collected, 2) AS total_copay_received,
    ROUND(total_patient_copay_liability - total_copay_collected, 2) AS outstanding_copay,
    
    -- Compliance requirement totals
    total_labs_required AS total_labs_ordered,
    total_labs_completed AS labs_performed,
    total_visits_required AS total_visits_ordered,
    total_visits_completed AS visits_completed,
    
    -- Performance percentages
    overall_utilization_pct AS pct_auth_utilized,
    revenue_realization_pct AS pct_revenue_captured,
    revenue_leakage_pct AS pct_revenue_lost,
    lab_compliance_pct AS pct_labs_completed,
    visit_compliance_pct AS pct_visits_completed,
    healthy_pipeline_pct AS pct_pipeline_healthy,
    
    -- Business intelligence alerts
    CASE 
        WHEN revenue_leakage_pct > 15 THEN 'CRITICAL_REVENUE_LEAKAGE_INVESTIGATE'
        WHEN expired_auths > active_auths THEN 'MORE_EXPIRED_THAN_ACTIVE_REVIEW_PROCESS'
        WHEN lab_compliance_pct < 80 THEN 'COMPLIANCE_RISK_PATIENT_SAFETY'
        WHEN revoked_auths > total_authorizations * 0.10 THEN 'HIGH_REVOCATION_RATE_FRAUD_CHECK'
        ELSE 'AUTHORIZATION_PORTFOLIO_HEALTHY'
    END AS portfolio_health_status,
    
    -- Data integrity check
    CASE 
        WHEN total_units_authorized <> units_filled_to_date + units_still_available 
        THEN 'QUANTITY_RECONCILIATION_ERROR'
        WHEN ABS(total_contract_value - (total_realized_revenue + total_pending_revenue)) > 1
        THEN 'FINANCIAL_RECONCILIATION_ERROR'
        ELSE 'TOTALS_BALANCED'
    END AS data_integrity_status
    
FROM grand_totals

ORDER BY 
    CASE WHEN manufacturer = 'ALL_MANUFACTURERS' THEN 0 ELSE 1 END,
    total_contract_value DESC;
```

**Workflow Explanation**:

1. **Status Aggregation** - Sums authorizations by status (active, expired, revoked, pending)
2. **Quantity Conservation** - Verifies authorized = dispensed + remaining (complete tracking)
3. **Financial Rollup** - Calculates contract value, realized revenue, and pending pipeline
4. **Compliance Totals** - Sums required vs completed labs and visits (patient safety tracking)
5. **Revenue Leakage** - Identifies expired + revoked as lost revenue (business impact)

-- TOTALS QUERY: Multi-dimensional sales analysis with complete component tracking
WITH base_totals AS (
    -- Calculate base totals across all dimensions
    SELECT 
        -- Temporal grouping
        DATE(sale_timestamp) AS sale_date,
        
        -- Product dimension attributes
        therapeutic_class,
        is_brand,
        is_controlled,
        
        -- Payer dimension
        primary_payer_type,
        
        -- Location dimension
        pharmacy_state,
        
        -- Core quantity totals
        COUNT(*) AS transaction_count,
        SUM(quantity_dispensed) AS total_units_sold,
        SUM(days_supply) AS total_days_supplied,
        
        -- Financial component totals (complete decomposition)
        SUM(ingredient_cost) AS total_ingredient_cost,
        SUM(dispensing_fee) AS total_dispensing_fees,
        SUM(sales_tax) AS total_sales_tax,
        SUM(copay_amount) AS total_copays,
        SUM(deductible_amount) AS total_deductibles,
        SUM(coinsurance_amount) AS total_coinsurance,
        SUM(plan_paid_amount) AS total_plan_payments,
        SUM(total_patient_pay) AS total_patient_responsibility,
        
        -- Discount component totals (often missed)
        SUM(coupon_discount) AS total_coupon_discounts,
        SUM(loyalty_discount) AS total_loyalty_discounts,
        SUM(cash_discount) AS total_cash_discounts,
        SUM(coupon_discount + loyalty_discount + cash_discount) AS total_all_discounts,
        
        -- Net revenue totals
        SUM(gross_revenue) AS total_gross_revenue,
        SUM(net_revenue) AS total_net_revenue,
        SUM(total_reimbursement) AS total_reimbursement,
        
        -- Verification: Gross should equal ingredient + fee + tax
        SUM(ingredient_cost + dispensing_fee + sales_tax) AS calculated_gross,
        
        -- Verification: Net should equal gross - discounts
        SUM(gross_revenue - (coupon_discount + loyalty_discount + cash_discount)) AS calculated_net
        
    FROM detailed_retail_sales
    
    GROUP BY 
        DATE(sale_timestamp),
        therapeutic_class,
        is_brand,
        is_controlled,
        primary_payer_type,
        pharmacy_state
),
hierarchical_rollups AS (
    -- Create hierarchical totals (drill-up paths)
    SELECT 
        sale_date,
        
        -- Level 1: All dimensions (most granular)
        therapeutic_class,
        is_brand,
        is_controlled,
        primary_payer_type,
        pharmacy_state,
        'L1_DETAIL' AS hierarchy_level,
        transaction_count,
        total_units_sold,
        total_net_revenue,
        total_all_discounts
        
    FROM base_totals
    
    UNION ALL
    
    -- Level 2: Roll up state (regional totals)
    SELECT 
        sale_date,
        therapeutic_class,
        is_brand,
        is_controlled,
        primary_payer_type,
        'ALL_STATES' AS pharmacy_state,
        'L2_REGIONAL' AS hierarchy_level,
        SUM(transaction_count),
        SUM(total_units_sold),
        SUM(total_net_revenue),
        SUM(total_all_discounts)
    FROM base_totals
    GROUP BY sale_date, therapeutic_class, is_brand, is_controlled, primary_payer_type
    
    UNION ALL
    
    -- Level 3: Roll up payer (product totals across payers)
    SELECT 
        sale_date,
        therapeutic_class,
        is_brand,
        is_controlled,
        'ALL_PAYERS' AS primary_payer_type,
        'ALL_STATES' AS pharmacy_state,
        'L3_PRODUCT' AS hierarchy_level,
        SUM(transaction_count),
        SUM(total_units_sold),
        SUM(total_net_revenue),
        SUM(total_all_discounts)
    FROM base_totals
    GROUP BY sale_date, therapeutic_class, is_brand, is_controlled
    
    UNION ALL
    
    -- Level 4: Grand total (all dimensions)
    SELECT 
        sale_date,
        'ALL_CLASSES' AS therapeutic_class,
        NULL AS is_brand,
        NULL AS is_controlled,
        'ALL_PAYERS' AS primary_payer_type,
        'ALL_STATES' AS pharmacy_state,
        'L4_GRAND' AS hierarchy_level,
        SUM(transaction_count),
        SUM(total_units_sold),
        SUM(total_net_revenue),
        SUM(total_all_discounts)
    FROM base_totals
    GROUP BY sale_date
),
integrity_validation AS (
    SELECT 
        *,
        
        -- Financial integrity checks
        CASE 
            WHEN hierarchy_level = 'L1_DETAIL' AND ABS(total_net_revenue - (SELECT total_net_revenue FROM base_totals b 
                WHERE b.sale_date = hr.sale_date AND b.therapeutic_class = hr.therapeutic_class 
                AND b.is_brand = hr.is_brand AND b.is_controlled = hr.is_controlled
                AND b.primary_payer_type = hr.primary_payer_type AND b.pharmacy_state = hr.pharmacy_state)) > 0.01
            THEN 'ROLLUP_MISMATCH'
            ELSE 'INTEGRITY_OK'
        END AS rollup_validation,
        
        -- Component completeness check
        CASE 
            WHEN total_all_discounts > total_net_revenue * 0.50 THEN 'DISCOUNT_EXCEEDS_50PCT_VERIFY'
            WHEN total_units_sold = 0 AND transaction_count > 0 THEN 'ZERO_UNITS_WITH_TRANSACTIONS'
            ELSE 'COMPONENTS_VALID'
        END AS component_check
        
    FROM hierarchical_rollups hr
)
SELECT 
    sale_date,
    hierarchy_level AS reporting_level,
    COALESCE(therapeutic_class, 'All Classes') AS therapeutic_category,
    COALESCE(CASE WHEN is_brand THEN 'Brand' WHEN NOT is_brand THEN 'Generic' END, 'All Types') AS product_type,
    COALESCE(CASE WHEN is_controlled THEN 'Controlled' ELSE 'Non-Controlled' END, 'All') AS control_status,
    COALESCE(primary_payer_type, 'All Payers') AS payer_segment,
    COALESCE(pharmacy_state, 'All States') AS geographic_region,
    
    -- Volume totals
    transaction_count AS total_transactions,
    total_units_sold AS total_units_dispensed,
    
    -- Financial totals (complete picture)
    ROUND(total_net_revenue, 2) AS net_revenue_total,
    ROUND(total_all_discounts, 2) AS total_discounts_given,
    
    -- Per-transaction averages (derived from totals)
    ROUND(total_net_revenue / NULLIF(transaction_count, 0), 2) AS avg_revenue_per_transaction,
    ROUND(total_units_sold * 1.0 / NULLIF(transaction_count, 0), 1) AS avg_units_per_transaction,
    
    -- Data quality
    rollup_validation AS hierarchical_integrity,
    component_check AS component_validation,
    
    -- Business alerts (silent failure detection)
    CASE 
        WHEN transaction_count > 1000 AND total_units_sold / transaction_count < 1 
        THEN 'SUSPICIOUS_LOW_UNITS_PER_RX_INVESTIGATE'
        WHEN total_all_discounts > total_net_revenue * 0.30 
        THEN 'HIGH_DISCOUNT_PCT_MARGINS_AT_RISK'
        WHEN hierarchy_level = 'L4_GRAND' AND total_transactions < (
            SELECT SUM(transaction_count) FROM hierarchical_rollups WHERE hierarchy_level = 'L1_DETAIL'
        )
        THEN 'GRAND_TOTAL_ROLLUP_ERROR'
        ELSE 'TOTALS_VALID'
    END AS business_validation_alert
    
FROM integrity_validation

WHERE hierarchy_level IN ('L1_DETAIL', 'L4_GRAND') -- Show detail and grand total for clarity

ORDER BY 
    sale_date,
    CASE hierarchy_level 
        WHEN 'L4_GRAND' THEN 0 
        WHEN 'L3_PRODUCT' THEN 1 
        WHEN 'L2_REGIONAL' THEN 2 
        ELSE 3 
    END,
    total_net_revenue DESC;
```

**Workflow Explanation**:

1. **Component Decomposition** - Sums every financial field separately (complete cost breakdown)
2. **Derived Totals** - Calculates gross, net, and reimbursement from components (verification)
3. **Hierarchical Rollups** - Creates L1 (detail) through L4 (grand total) levels (drill-down capability)
4. **Conservation Checks** - Verifies ingredient + fee + tax = gross (financial conservation)
5. **Union Aggregation** - Uses UNION ALL to combine different grouping levels (complete hierarchy)

## Key Totals Concepts Summary

| Total Type | Calculation Method | Pharma Use Case | Integrity Check |
|-----------|-------------------|-----------------|-----------------|
| **Simple Sum** | `SUM(column)` | Total claims, total units | Cross-foot with count |
| **Conditional Sum** | `SUM(CASE WHEN...)` | Paid claims only, active inventory | Verify filter logic |
| **Component Sum** | Sum of parts | Financial reconciliation | Parts = Whole |
| **Hierarchical Rollup** | UNION ALL levels | Drill-down reporting | Child sums = Parent |
| **Running Total** | `SUM() OVER (ORDER BY)` | Cumulative sales, year-to-date | Reset at boundaries |
| **Distinct Sum** | `SUM(DISTINCT...)` | Unique patient counts | Avoid double-counting |

**Critical Totals Validation Rules**:

1. **Conservation Laws** - Patient Pay + Plan Pay = Allowed Amount (payment conservation)
2. **Hierarchical Integrity** - Sum of child totals must equal parent total (rollup validation)
3. **Component Completeness** - All cost components must be identified (no hidden fees)
4. **Temporal Consistency** - Running totals must reset at appropriate boundaries (day/month/year)
5. **Status Reconciliation** - Active + Expired + Revoked = Total (state conservation)

**Silent Failure Detection in Totals**:

| Warning Sign | Detection Method | Business Impact |
|-------------|----------------|-----------------|
| **Missing Components** | Reconstruction != Reported | Understated costs |
| **Rounding Errors** | Variance > $0.01 | Financial imbalance |
| **Double Counting** | DISTINCT count < SUM count | Inflated revenue |
| **Orphan Records** | Child totals > Parent | Data integrity failure |
| **Stale Totals** | Last update > 24 hours | Decisions on old data |

## Key Points
- Totals are calculated by summing all relevant quantities or values.
- Inclusion of all components is crucial for an accurate total.
- Totals can be affected by factors such as taxes, discounts, and additional fees.
- Regular review and update of totals are necessary for ongoing accuracy.
- Automation, such as using spreadsheet formulas, can help in calculating and maintaining totals.

> [!important]
> The total must always be recalculated whenever any of its components change to maintain accuracy.

## Mental Model
1. **Identification**: Identify all components that contribute to the total.
2. **Calculation**: Sum all identified components, considering factors like taxes and discounts.
3. **Review and Update**: Regularly review the total for accuracy and update it as necessary.

## Observe
- A common mistake people make is forgetting to include all relevant components when calculating totals.
- A silent failure mode can occur when totals are not regularly updated, leading to outdated and inaccurate information.
- A misleading assumption is that totals remain static; they can change frequently due to various factors.
